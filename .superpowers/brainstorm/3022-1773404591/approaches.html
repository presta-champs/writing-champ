Spending Dashboard — Approach

How should the dashboard be structured within the app?

A

Dedicated /dashboard/usage page (Recommended)

New sidebar nav item "Usage" between Library and Settings. Single page with sections: spending summary cards at top, daily chart in the middle, breakdowns (by model, by event type, by user) below, and budget settings at the bottom.

Pros

  • Clean separation — doesn't clutter existing pages
  • Room for charts, tables, and drill-downs
  • Follows existing pattern (each nav item = one page)

Cons

  • One more nav item
  • User must navigate to see spending
B

Embedded in Settings page

Add a "Usage & Billing" section to the existing Settings page. Spending summary, chart, and budget controls all live alongside org settings, API keys, and editorial guidelines.

Pros

  • No new nav item
  • Budget settings near other org config

Cons

  • Settings page is already dense
  • Charts + tables make it very long
  • Harder to add detail views later
C

Dashboard home widget + dedicated page

Add a small "This month's spend" card to the dashboard home (alongside Sites, Personas, Articles), and clicking it opens a full /dashboard/usage page with all the detail.

Pros

  • Spending visible at a glance on home
  • Full detail available one click away
  • Budget warning visible without navigating

Cons

  • Two places to build (widget + full page)
  • Slightly more work
